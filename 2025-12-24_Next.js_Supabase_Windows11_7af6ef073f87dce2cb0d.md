## 考察

- **oxlint**: 圧倒的な高速化を確認しました。CIでのLintチェック時間を劇的に短縮できます。
- **oxfmt / tsgo**: こちらも5倍以上の高速化を達成しており、ローカル開発時の待ち時間短縮に寄与します。
- **検出精度**: 測定中に `src/components/works/work-registration/work-registration.tsx` の構文エラーが存在しましたが、既存ツール・新ツール共に正しくエラー（終了コード1）を返しました。エラー検出能力も機能しています。

## 今後の運用

`package.json` に以下のコマンドを追加済みです。CIやpre-commit hookでの活用を推奨します。

- `npm run lint:fast`
- `npm run format:fast`
- `npm run typecheck:fast`

## 型チェックのオプション

```script
    "lint:fast": "oxlint --tsconfig ./tsconfig.json --type-aware --type-check ./src",
    "format:fast": "oxfmt --write .",
    "typecheck:fast": "tsgo --noEmit"
```

※lint:fast に 型チェックのオプションを追加しています。

oxlint に `--type-check` オプションを渡すと、内部的に tsgo（正確には oxlint-tsgolint というブリッジパッケージ）の機能を利用して型チェックを行います。

つまり、oxlint コマンド一つを実行するだけで、Rust製の高速なリントと、tsgo による高速な型チェックの両方をまとめて行えるようになります。これにより、個別に `run typecheck:fast` を実行する手間が省け、CIの時間も短縮できます。
