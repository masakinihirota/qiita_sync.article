### 本文のドラフト

**はじめに**
日々の開発でAIアシスタントは手放せない存在ですが、使っているとどうしても直面するのが「クォータ（利用枠）の枯渇」問題です。
私の場合は、**GitHub Copilot**と**Antigravity**（それぞれ別のプランに加入）を併用し、一方の利用枠がなくなったらもう一方へ切り替えるという運用で乗り切っています。

**直面した課題：設定ファイルの二重管理**
この併用運用をしていると、ひとつ問題が発生しました。
それは、スラッシュコマンドやSkill（スキル）ファイルなどの設定が、ツールごとに別々のフォルダで管理されてしまうことです。新しいコマンドを追加したり修正したりするたびに、両方のフォルダを更新しなければならず、非常に手間がかかっていました。

**解決策：AntigravityのフォルダにCopilotを合わせる**
この二重管理を解消するため、両方のツールで同じフォルダを参照するように設定を共通化しました。

ポイントは以下の通りです：

* **Antigravity側**: 参照先のフォルダが固定されているため、デフォルトの仕様をそのまま使います。
* **GitHub Copilot側**: 参照先のフォルダを柔軟に設定できるため、Antigravityのフォルダを読みに行くように設定を変更します。

**具体的な設定手順**
VS Codeの `settings.json` を開き、GitHub Copilot側の設定として以下のコードを追記します。

```json
    // スラッシュコマンドの共通化（GitHub CopilotとAntigravity）
    "chat.promptFilesLocations": {
        ".agent/workflows": true
    },
    // skillsの場所を指定する設定
    "chat.agentSkillsLocations": {
        ".agent/skills": true
    }

```

この設定により、ディレクトリ構成は以下のようになり、1つのフォルダ（`.agent/`）で両方のAIツールの設定を賄えるようになります。

```text
📁 プロジェクトルート
 └── 📁 .agent
      ├── 📁 workflows  ← 両方のツールがスラッシュコマンドとして参照
      └── 📁 skills     ← 両方のツールがスキルファイルとして参照

```

**おわりに**
設定ファイルの場所を統一するちょっとした工夫ですが、これでどちらのツールを使っている時でも同じコマンドが呼び出せるようになり、開発体験が劇的に向上しました。

これより下をAI(gemini3)に丸投げ

GitHub CopilotとAntigravityはそれぞれ別のプランに入りクォータがなくなると切り替えて使っています。
ここで問題なのが共通のファイルがスラッシュコマンドやskillファイルなどを使うと別々のフォルダで管理していました。

Antigravity側では固定ですが、GitHub Copilotではそれぞれ設定できるので共通化して一元管理するようにしました。

スラッシュコマンドがGitHub CopilotをAntigravityで別々に管理していました。

Antigravity側はデフォルトの設定を使います。

GitHub Copilot側で下記のように設定します。

```settings.json
    // スラッシュコマンドの共通化 GitHub CopilotとAntigravity
	"chat.promptFilesLocations": {
		".agent/workflows": true
	},
	// skillsの場所を指定する設定
	"chat.agentSkillsLocations": {
		".agent/skills": true
	},

```
