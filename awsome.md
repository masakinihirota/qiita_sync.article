# Codex 用 ExecPlan テンプレート

AGENTS.md と ExecPlan のマークダウンテンプレートは、AI コーディングアシスタントと効率的にプログラミングするためのものです。

**「ExecPlan」** は、あなたとエージェントの共通語となり、機能をどのように構築するかを構造化するタスク単位の作業計画を指すコードワードです。

**[Aaron Friel](https://x.com/AaronFriel) の OpenAI 2025 DevDay プレゼンテーション「Shipping with Codex」にインスパイアされました：** [https://youtu.be/Gr41tYOzE20?t=1050](https://youtu.be/Gr41tYOzE20?t=1050)

メンテナー: [Aditya Advani](https://x.com/aditya_advani)

---

## ワークフロー

1. **`.agents/template/PLAN.md` を作成** — 機能の計画のための構造化されたマークダウンテンプレートを用意します。
2. **`AGENTS.md` を設定** — AI エージェントにテンプレートをコピーして各タスクの ExecPlan を生成するよう指示します（例: `.agents/orchestrator/PLAN.md`）。
3. **「do the plan」と伝える** — エージェントは順序立てて実行し、通常 1 回〜2〜3 回の反復で完了します。

### なぜうまくいくのか

- **構造化された思考:** エージェント（とあなた）に目的、コンテキスト、手順を明確化させます。
- **進化するドキュメント:** 進捗、意思決定、想定外の事象は計画書に残ります。
- **クリーンな作業領域:** 各機能は独自のフォルダを持ち、マージ後に削除するオプションがあります。
- **再現性:** テンプレートには TDD、意思決定ログ、検証手順などのベストプラクティスが組み込まれています。

---

## クイックスタート

1. このリポジトリの `.agents/template/PLAN.md` と `AGENTS.md` をプロジェクトにコピーします。
2. **(任意)** 計画をローカルに保持する場合は `.gitignore` に追加します:
   ```
   .agents/*
   !.agents/template/
   ```
3. エージェントに伝える: 「テンプレートに従って [機能名] の ExecPlan を作成して」
4. 実行: 「Do the plan」

---

## 使用上の注意

- **一つの ExecPlan = 一つのタスク:** 各プランは機能、リファクタ、バグ修正などの一つのタスクに対応します。進捗セクションでサブタスクを追跡することもあります。
- **TDD を最優先:** `AGENTS.md` にはテスト駆動開発の指針が含まれています。
- **並行実行:** 複数のプランを並行して実行することは可能ですが、オーケストレーションが必要です（こちらはまだ試行中です）。
- **クリーンアップ:** 共有コードベースで作業する場合は、マージ後に機能フォルダを削除してください。
- **細かい差異:** 動画では Aaron が `.agent` を使っていますが、私は `AGENTS.md` の名前に合わせて `.agents` を使っています。つい忘れます。
