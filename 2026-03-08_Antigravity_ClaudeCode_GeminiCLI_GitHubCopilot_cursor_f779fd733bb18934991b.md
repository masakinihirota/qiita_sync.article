## 1. 導入：スキルの手動配置（インストール）

Impeccableは、GitHubリポジトリから「スキル（機能の実体）」を直接環境に読み込ませる方式を推奨しています。

### 手順

1. [紹介ページ](https://impeccable.style/) からZipファイルをダウンロードします。

「Prefix with /i-」 をONにするとskillのファイル名に 「i-」が付きます。(例:i-extract) skillsをパッケージ単位で管理したい場合や既存のskillsのファイル名が被ってる場合に便利です。
2. ファイルを解凍し、内部にある `skills` フォルダをコピーします。
3. 使用しているエディタ（Cursor等）やCLI実行環境のルートにある **`skills` フォルダ** へ、解凍した内容を貼り付けてください。
4. これにより、エディタのチャット欄で `/` を入力した際、各検証ロジックが「スキル」として認識され、実行可能になります。

`.agentsフォルダ`内に置く場合の設定

設定ファイル VSCode

```setting.json
	// 指示書ファイルの場所を指定する設定
	"chat.instructionsFilesLocations": {
		".agents/rules": true
	},
	// 「skills」の場所を指定する設定
	"chat.agentSkillsLocations": {
		".agents/skills": true
	},
    // プロンプトファイルの場所を指定する設定
	"chat.promptFilesLocations": {
		".agents/workflows": true
	},
	// フックファイルの場所を指定する設定
	"chat.hookFilesLocations": {
		".agents/hooks": true
	},

```

## 2. 開発フェーズ別：全17コマンド完全リファレンス

開発の進行状況に合わせて、必要なコマンドをステップバイステップで紹介します。

### 【STEP 1】診断と基盤構築（Diagnostic & System）

プロジェクトの現状を分析し、AIにコンテキストを学習させるフェーズです。

* **`/teach-impeccable`**
初回のみ1回だけの現状分析を行います。
プロジェクトのデザインコンテキストを収集し、AI設定ファイルに保存します。導入時に一度実行することで、以降の提案精度を向上させます。
一番初めに使います、それぞれ質問が10回ほどあってすべて答えるとAGENTS.mdなど指示書にプロンプトが書かれます。

* **`/critique`**
UXの観点からデザインを評価します。視覚的階層、情報設計、感情的な響きなどを分析し、具体的なフィードバックを提供します。

### 【STEP 2】品質の向上と適応（Quality & Adaptation）

実装が進む中で、インターフェースを堅牢にし、各環境に最適化するフェーズです。

* **`/normalize`**
デザインシステムに適合させ、全体の整合性と一貫性を確保します。
* **`/optimize`**
読み込み速度、レンダリング、画像、バンドルサイズを改善し、体験を高速化します。
* **`/adapt`**
スマホ、PC等異なる画面サイズ、デバイス、コンテキストに合わせてデザインを適応させます。
* **`/clarify`**
不明瞭なUXコピー、エラーメッセージ、ラベルを改善し、ユーザーの理解を助けます。
* **`/distill`**
不要な複雑さを削ぎ落とし、デザインを本質的でクリーンな状態に凝縮します。

### 【STEP 3】デザインの洗練と演出（Intensity & Enhancement）

リリース直前に、デザインのトーンを調整し、ユーザー体験を魅力的にするフェーズです。


* **`/bolder`**
保守的すぎるデザインに視覚的インパクトを加え、より印象的にします。
* **`/quieter`**
主張が強すぎるデザインのトーンを抑え、バランスを整えます。
* **`/animate`**
使いやすさと喜びを向上させる、目的を持ったアニメーションやマイクロインタラクションを追加します。
* **`/colorize`**
単調な機能に対し、戦略的に色を加えて表現力を高めます。
* **`/delight`**
ユーザーの記憶に残る、喜びや個性を感じさせる演出を加えます。

### 【STEP 4】システム化（System）

成果を資産化し、継続的な管理を容易にするフェーズです。

* **`/extract`**
再利用可能なコンポーネントやデザイントークンを抽出し、デザインシステムへ統合します。
* **`/onboard`**
オンボーディングフローや空の状態（Empty States）を改善し、ユーザーが価値をすぐに理解できるようにします。


### 【STEP 5】最終段階(finally)

* **`/audit`**
アクセシビリティ、パフォーマンス、テーマ、レスポンシブデザインを網羅的に監査します。問題の重要度と推奨策を含む詳細なレポートを生成します。

* **`/polish`**
最終段階前の最終調整。配置、余白、一貫性など、細部を修正して完成度を高めます。

* **`/harden`**
エラーハンドリング、多言語対応（i18n）、テキスト溢れ、エッジケースへの対応を強化し、堅牢なプロダクトにします。
予期せぬエラーで止まらないように、コードの耐久性を高めます。

## まとめ：AIと共創する高品質なWebサイト

Impeccableは、`skills` フォルダを配置するだけで、AIエディタから**指示**や**スラッシュコマンド**一つでこれらの高度な機能を引き出せます。公式の17コマンドを適切に使い分け、Web標準に準拠した高品質なプロダクトを構築しましょう。
